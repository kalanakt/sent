# TL Reference

**sent** implements Telegram's MTProto layer 225. The TL (Type Language) layer provides typed constructors for every Telegram API object and RPC method.

## Imports

```python
from sent import types, functions

# Or import from namespace modules directly:
from sent.tl.types.messages import Message
from sent.tl.functions.messages import MessagesSendMessage
```

## Raw RPC

Use `TelegramClient` as a callable to invoke any TL request:

```python
from sent import TelegramClient, functions

async with TelegramClient("session", API_ID, API_HASH) as client:
    await client.start()
    peer = await client.get_input_entity("username")
    result = await client(
        functions.messages.MessagesGetHistory(
            peer=peer,
            offset_id=0,
            offset_date=0,
            add_offset=0,
            limit=10,
            max_id=0,
            min_id=0,
            hash=0,
        )
    )
```

## When to use raw TL

| Use high-level client methods | Use raw TL |
|-------------------------------|------------|
| Sending messages, files, dialogs | New or niche API methods not wrapped yet |
| Entity resolution | Custom flag combinations on RPC calls |
| Most bot and user workflows | Low-level auth or transport debugging |

See the [Raw TL guide](/guides/raw-tl) for patterns and the [Client API](/api/client) for wrapped methods.
