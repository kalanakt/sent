# Build with AI

**sent** docs are written for both humans and AI agents. Point your coding assistant at these pages to scaffold bots, handlers, and integrations without guessing the API.

## Quick prompt

You can get started by asking your AI agent to read the docs — same pattern as [Vocs](https://vocs.dev):

### Root page

```
Read https://sentdoc.vercel.app/ and build a Telegram bot using sent.
```

### Root page + full index

```
Read https://sentdoc.vercel.app/llms-full.txt and build a Telegram bot using sent.
Follow https://sentdoc.vercel.app/guides/bot and https://sentdoc.vercel.app/guides/events.
Use https://sentdoc.vercel.app/api/client for method signatures.
```

```

Follow https://sentdoc.vercel.app/guides/bot and https://sentdoc.vercel.app/guides/events.

```

## What to tell the agent

Include these constraints for better results:

- **Library:** `sent` (async MTProto client), not python-telegram-bot or Telethon
- **Python:** 3.8+, use `asyncio` and `await client.start(bot_token=...)`
- **Credentials:** never hardcode tokens — use environment variables
- **Examples:** see [examples/echo_bot.py](https://github.com/kalanakt/sent/blob/main/examples/echo_bot.py) in the repo

## Run docs locally

For editing docs or testing changes before deploy:

```bash
cd docs
bun install
bun run dev    # http://localhost:5173
```

Use `https://sentdoc.vercel.app/llms-full.txt` in prompts for the published site, or swap the host to `http://localhost:5173` when testing against your local dev server.
